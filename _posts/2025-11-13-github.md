# Guide d'utilisation de Git et GitHub

Ce guide vous explique comment installer, configurer et utiliser Git et GitHub pour gérer votre code source.

---

### Étape 1 : Installer Git

Avant de pouvoir utiliser Git, vous devez l'installer sur votre ordinateur.

1.  Rendez-vous sur [git-scm.com](https://git-scm.com/) pour télécharger la version de Git adaptée à votre système d'exploitation (Windows, macOS ou Linux).
2.  Exécutez le programme d'installation et suivez les instructions. Les options par défaut sont généralement suffisantes pour la plupart des utilisateurs.

---

### Étape 2 : Configurer Git

Une fois Git installé, vous devez configurer votre identité. Ces informations seront associées à tous les commits que vous créerez.

1.  Ouvrez un terminal (PowerShell ou Git Bash sur Windows, Terminal sur macOS et Linux).
2.  Configurez votre nom d'utilisateur et votre adresse e-mail avec les commandes suivantes :

    ```bash
    git config --global user.name "Votre Nom"
    git config --global user.email "votre.email@example.com"
    ```

3.  Vous pouvez vérifier que la configuration a bien été appliquée avec la commande :

    ```bash
    git config --list
    ```

---

### Étape 3 : S'authentifier auprès de GitHub

Pour interagir avec vos dépôts sur GitHub, vous devez vous authentifier. Il existe deux méthodes principales : HTTPS et SSH.

#### Option A : HTTPS (Recommandé pour les débutants)

L'authentification HTTPS est plus simple à configurer.

1.  Si vous n'en avez pas, créez un compte sur [github.com](https://github.com/).
2.  Lorsque vous effectuerez une opération comme `git push` ou `git clone` sur un dépôt privé, Git vous demandera votre nom d'utilisateur GitHub et un **token d'accès personnel (Personal Access Token)**.
3.  Pour créer un token :
    *   Allez dans les **paramètres** de votre compte GitHub.
    *   Accédez à **Developer settings** > **Personal access tokens**.
    *   Générez un nouveau token en lui donnant les permissions nécessaires (par exemple, `repo`).
    *   **Copiez ce token et conservez-le en lieu sûr**, car il ne sera plus visible. Utilisez-le comme mot de passe dans le terminal.

#### Option B : SSH (Plus sécurisé et pratique)

L'authentification SSH utilise un système de clés cryptographiques pour établir une connexion sécurisée.

1.  Générez une nouvelle clé SSH sur votre machine :

    ```bash
    ssh-keygen -t ed25519 -C "votre.email@example.com"
    ```

2.  Ajoutez la clé à l'agent SSH pour ne pas avoir à saisir votre mot de passe à chaque fois :

    ```bash
    eval "$(ssh-agent -s)"
    ssh-add ~/.ssh/id_ed25519
    ```

3.  Copiez votre clé SSH publique. Le contenu du fichier `id_ed25519.pub` doit être copié :

    ```bash
    cat ~/.ssh/id_ed25519.pub
    ```

4.  Ajoutez cette clé à votre compte GitHub :
    *   Allez dans les **paramètres** de votre compte GitHub.
    *   Accédez à **SSH and GPG keys**.
    *   Cliquez sur **New SSH key**, donnez-lui un titre et collez votre clé publique.

### Étape 4 : Créer un dépôt et pousser du code

Voici comment créer un nouveau projet localement et le publier sur GitHub.

1.  Sur votre machine, créez un nouveau dossier, initialisez un dépôt Git et ajoutez un premier fichier :

    ```bash
    mkdir mon-projet
    cd mon-projet
    git init
    echo "# Mon Super Projet" > README.md
    git add README.md
    git commit -m "Premier commit : ajout du README"
    ```

2.  Allez sur [github.com/new](https://github.com/new) et créez un nouveau dépôt. **Ne cochez pas** l'option pour initialiser le dépôt avec un `README.md` ou un `.gitignore`, car nous avons déjà notre projet en local.

3.  Copiez l'URL de votre nouveau dépôt (par exemple, `https://github.com/votre-nom/mon-projet.git`).

4.  Connectez votre dépôt local au dépôt distant sur GitHub et poussez votre code :

    ```bash
    git remote add origin https://github.com/votre-nom/mon-projet.git
    git branch -M main
    git push -u origin main
    ```

    Votre code est maintenant en ligne sur GitHub.

---

### Étape 5 : Cloner un dépôt existant

Pour récupérer un projet qui existe déjà sur GitHub, vous devez le "cloner".

1.  Sur la page du dépôt GitHub, cliquez sur le bouton vert **Code** et copiez l'URL (HTTPS ou SSH).
2.  Dans votre terminal, naviguez jusqu'à l'endroit où vous souhaitez enregistrer le projet, puis exécutez la commande `git clone`.

    **Exemple avec HTTPS :**

    ```bash
    git clone https://github.com/kluster/dirty-desktop.git
    ```

    **Exemple avec SSH :**

    ```bash
    git clone git@github.com:kluster/dirty-desktop.git
    ```

    Cette commande créera un dossier nommé `dirty-desktop` contenant tous les fichiers du projet.

### Opérations courantes

#### Mettre à jour son dépôt distant

Après avoir modifié votre code localement, suivez ces étapes pour envoyer vos modifications sur GitHub :

1.  Ajoutez les fichiers modifiés à la "staging area" :

    ```bash
    # Pour ajouter tous les fichiers modifiés
    git add .

    # Pour ajouter un fichier spécifique
    git add nom-du-fichier.js
    ```

2.  Créez un "commit" pour sauvegarder vos modifications avec un message descriptif :

    ```bash
    git commit -m "Ajout de la fonctionnalité X"
    ```

3.  Poussez vos commits vers le dépôt distant sur GitHub :

    ```bash
    git push origin main
    ```

#### Changer la visibilité d'un dépôt

Vous pouvez changer la visibilité de votre dépôt (de public à privé, ou inversement) directement sur le site de GitHub.

1.  Accédez à votre dépôt sur GitHub.
2.  Cliquez sur l'onglet **Settings** (Paramètres).
3.  Dans la section **Danger Zone**, trouvez l'option **Change repository visibility**.
4.  Cliquez sur **Change visibility**, choisissez l'option souhaitée et suivez les instructions pour confirmer.
